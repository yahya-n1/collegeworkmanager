# College Work Manager

This will be a Desktop application to help students and professionals organize their calendars so they can better utilize their time as they juggle many activities - school, work, meetings, assignments - all of which are located in many different locations. 
The goal of our capstone project is to develop a web-based system that would integrate all of these various calendars into one simple user-friendly calendar application that allows for calendar creation, setting reminders, assigning tasks, and collaboration, within one calendar that may be used by all users to share calendars.
Users will have the option of creating personal calendars as well as creating shared group calendars for team projects or similar work environments. Users can add and edit events to their calendars, delete events, create and set reminders on a recurring basis and classify events by type (i.e. school, work, personal). Users will receive notifications of individual user event updates, colors associated with their events on their respective calendars, and have a conflict detection function to identify when a user may have overlapping events. Team members may share a group workspace and assign tasks to each other within a shared calendar.
This project directly addresses the issues of disorganization with scheduling and lack of effective communication and coordination among team members. The Smart Collaborative Calendar System will result in increased levels of productivity and improved time management by utilizing one platform to house task management, reminders and collaboration. This project illustrates full stack application development, database design and development, user authentication methodologies, and collaborative software engineering processes.

## World Assumptions
Access to computers with Internet connection.
Ability to share their calendars with multiple other people.
Functionality will occur in a realistic academic or business environment.
User will expect secure logon procedures.
User will expect privacy concerning their calendar activities.

## User Requirements
### Functional Requirements:
#### User must be able to:
- Create their account and log in to the system.
- Create, edit, and/or delete calendar events.
- Specify the event date, time, description, and category when they create an event.
- Set a reminder/notification for that event.
- View events based on daily, weekly, or monthly views.
- Share their calendars with other users.
- Accept/decline invitations to calendars that are shared with them by other users.
- Detect and alert the user to calendar conflicts.
- Search for calendar events based on keywords or event dates.

### Non-Functional Requirements:
- 3 seconds load time upon accessing the system.
- System should be able to accommodate at least 200 concurrent users.
- System is secure (passwords will be encrypted).
- System is user-friendly and intuitive to use.
- System works on all desktop systems.

## Specifications:
### System Requirements:
- Desktop Application.
- User-side Technologies: Java
- Server-side Technologies: Java (Spring Boot) OR Node.js (depending on your choice)
- Database: MySQL.
- Login Security: Username & password with encrypted password storage.

### Interface Specifications:
- Login screen
- Dashboard screen
- Calendar view screen.
